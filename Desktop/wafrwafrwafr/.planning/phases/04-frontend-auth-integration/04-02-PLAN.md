**Step 3 — Guard `/new-assessment` route for team users only:**

Read `aws-frontend/app/new-assessment/page.tsx`. At the top of the component:

Import `getCurrentUserInfo`, `isTeamUser` from `"@/lib/auth"`.
Import `useRouter` from `"next/navigation"` (likely already imported).

Add an effect that checks the user's role and redirects clients:
```typescript
const [authorized, setAuthorized] = useState(false);

useEffect(() => {
  getCurrentUserInfo()
    .then(info => {
      if (!isTeamUser(info.groups)) {
        router.push("/"); // Redirect clients to dashboard
      } else {
        setAuthorized(true);
      }
    })
    .catch(() => router.push("/"));
}, [router]);
```

Wrap the component's return JSX with: `if (!authorized) return null;` (or a loading spinner). This prevents a flash of the new-assessment form before the redirect happens.
