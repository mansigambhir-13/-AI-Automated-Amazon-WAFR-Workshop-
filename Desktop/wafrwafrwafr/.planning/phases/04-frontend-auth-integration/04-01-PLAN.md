**Step 1 — Update `lib/api.ts` to attach Bearer token:**

Import `getAccessToken` from `"./auth"`.

Add a private `authHeaders()` helper:
```typescript
async function authHeaders(): Promise<Record<string, string>> {
  try {
    const token = await getAccessToken();
    return { Authorization: `Bearer ${token}` };
  } catch {
    // If not authenticated (e.g., during initial load before Authenticator resolves), return empty.
    // The Authenticator component will redirect to login, and the backend will return 401.
    return {};
  }
}
```

Update `apiGet<T>`:
- Change `const res = await fetch(url);` to `const res = await fetch(url, { headers: await authHeaders() });`
- Add 401 handling: if `res.status === 401`, call `window.location.href = "/"` and throw `new Error("Session expired")`. This triggers a full page reload which re-enters the Authenticator gate. Use the try-refresh-then-redirect pattern: before redirecting, attempt `fetchAuthSession({ forceRefresh: true })` from `"aws-amplify/auth"`. If refresh succeeds, retry the request once. If it fails, then redirect.

Actually, simplify: Amplify's `fetchAuthSession()` already auto-refreshes tokens. If the backend still returns 401 after `getAccessToken()` provides a token, the refresh token itself is expired. In that case, redirect to `/` which forces re-login via Authenticator. Do NOT implement a retry loop — one redirect is enough.

Update `apiPost<T>`:
- Merge auth headers with content-type: `headers: { "Content-Type": "application/json", ...(await authHeaders()) }`
- Add same 401 redirect handling as apiGet.

Update `apiDelete<T>`:
- Change `const res = await fetch(url, { method: "DELETE" });` to `const res = await fetch(url, { method: "DELETE", headers: await authHeaders() });`
- Add same 401 redirect handling.

Export `authHeaders` (named export) so `sse-client.ts` can use it.

These three functions are the only places that hardcode `'frontend-user'`. After this change, the audit trail (Phase 3) will record the real Cognito username instead of a placeholder.

Additionally, update the `getReportDownloadUrl()`, `getAwsReportDownloadUrl()`, and `getResultsDownloadUrl()` functions. These return raw URLs used for download links — the browser navigates directly to them. Since the backend now requires auth, these downloads will fail without a token. Change these from returning a URL string to an async function that fetches the download with auth:

```typescript
export async function downloadReport(sessionId: string): Promise<void> {
  const headers = await authHeaders();
  const url = `${BACKEND_URL}/api/wafr/session/${sessionId}/report/download`;
  const res = await fetch(url, { headers });
  if (!res.ok) throw new Error(`Download failed: ${res.status}`);
  const blob = await res.blob();
  const a = document.createElement("a");
  a.href = URL.createObjectURL(blob);
  a.download = `wafr-report-${sessionId}.pdf`;
  a.click();
  URL.revokeObjectURL(a.href);
}
```

Apply the same pattern for `downloadAwsReport` and `downloadResults`. Import `authHeaders` from `"./api"`.

Keep the old `getReportDownloadUrl` etc. functions as well for backwards compatibility, but add the new `downloadReport` / `downloadAwsReport` / `downloadResults` async functions that callers should migrate to. Mark the old URL-returning functions with a `@deprecated` JSDoc comment.

Note: If report download pages use `<a href={getReportDownloadUrl(id)}>`, those must be updated to `onClick` handlers calling the new `downloadReport()`. Check `aws-frontend/app/reports/` for the download page and update any direct `href` usage.
