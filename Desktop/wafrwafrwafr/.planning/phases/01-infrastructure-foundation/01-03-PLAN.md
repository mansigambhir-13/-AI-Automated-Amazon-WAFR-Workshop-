  <name>Task 1: Extend IAM inline policy and create Secrets Manager secrets</name>
  <files></files>
  <action>
This task has two parts: IAM policy extension and Secrets Manager secret creation.

**Part A: Discover and extend the IAM inline policy**

Step 1 — Discover the current inline policy name:
```bash
aws iam list-role-policies --role-name WafrAppRunnerInstanceRole --region us-east-1
```
This returns the policy name(s). Use the first one (likely there is only one).

Step 2 — Read the current policy document:
```bash
aws iam get-role-policy --role-name WafrAppRunnerInstanceRole --policy-name <POLICY_NAME> --region us-east-1
```
Save the `PolicyDocument` JSON. This contains existing Bedrock, S3, WA Tool, and CloudWatch permissions that MUST be preserved.

Step 3 — Merge three new statements into the existing policy's Statement array:

Statement 1 — DynamoDB CRUD:
```json
{
  "Sid": "DynamoDBCRUD",
  "Effect": "Allow",
  "Action": [
    "dynamodb:GetItem", "dynamodb:PutItem", "dynamodb:UpdateItem",
    "dynamodb:DeleteItem", "dynamodb:Query", "dynamodb:Scan",
    "dynamodb:BatchGetItem", "dynamodb:BatchWriteItem", "dynamodb:DescribeTable"
  ],
  "Resource": [
    "arn:aws:dynamodb:us-east-1:842387632939:table/wafr-*",
    "arn:aws:dynamodb:us-east-1:842387632939:table/wafr-*/index/*"
  ]
}
```
IMPORTANT: Include both `table/wafr-*` AND `table/wafr-*/index/*` — GSI queries require the index path (Pitfall 4 from research).

Statement 2 — Cognito read-only:
```json
{
  "Sid": "CognitoReadOnly",
  "Effect": "Allow",
  "Action": [
    "cognito-idp:AdminGetUser",
    "cognito-idp:ListUsersInGroup",
    "cognito-idp:DescribeUserPool"
  ],
  "Resource": "arn:aws:cognito-idp:us-east-1:842387632939:userpool/*"
}
```

Statement 3 — Secrets Manager read for Cognito secrets:
```json
{
  "Sid": "SecretsManagerCognitoRead",
  "Effect": "Allow",
  "Action": ["secretsmanager:GetSecretValue"],
  "Resource": "arn:aws:secretsmanager:us-east-1:842387632939:secret:wafr-cognito-*"
}
```
Uses wildcard suffix because Secrets Manager appends a random 6-char suffix to ARNs (Pitfall 6 from research).

Step 4 — Write the merged policy back:
```bash
aws iam put-role-policy \
  --role-name WafrAppRunnerInstanceRole \
  --policy-name <POLICY_NAME> \
  --policy-document file://extended-policy.json \
  --region us-east-1
```

**Part B: Create Secrets Manager secrets for Cognito values**

Read the Cognito Pool ID and Client ID from Plan 01-02 SUMMARY.

Create two separate secrets (not one combined JSON — App Runner maps one secret ARN to one env var):
```bash
aws secretsmanager create-secret \
  --name wafr-cognito-user-pool-id \
  --description "WAFR Cognito User Pool ID" \
  --secret-string "<POOL_ID>" \
  --region us-east-1

aws secretsmanager create-secret \
  --name wafr-cognito-client-id \
  --description "WAFR Cognito App Client ID" \
  --secret-string "<CLIENT_ID>" \
  --region us-east-1
```

Capture both secret ARNs from the responses — needed for Task 2 App Runner env var configuration.

Update both App Runner services to inject DynamoDB table names, Cognito secret references, and AUTH_REQUIRED flag.

CRITICAL: App Runner `update-service` REPLACES all environment variables — it does NOT merge (Pitfall 1 from research). You MUST read the current configuration first and include ALL existing variables in the update.

**Step 1 — Discover service ARNs:**
```bash
BACKEND_ARN=$(aws apprunner list-services --region us-east-1 \
  --query "ServiceSummaryList[?ServiceUrl=='i5kj2nnkxd.us-east-1.awsapprunner.com'].ServiceArn" --output text)

FRONTEND_ARN=$(aws apprunner list-services --region us-east-1 \
  --query "ServiceSummaryList[?ServiceUrl=='3fhp6mfj7u.us-east-1.awsapprunner.com'].ServiceArn" --output text)
```

**Step 2 — Read current env vars for BOTH services:**
```bash
aws apprunner describe-service --service-arn $BACKEND_ARN --region us-east-1 \
  --query "Service.SourceConfiguration.ImageRepository.ImageConfiguration"

aws apprunner describe-service --service-arn $FRONTEND_ARN --region us-east-1 \
  --query "Service.SourceConfiguration.ImageRepository.ImageConfiguration"
```

Save the current `RuntimeEnvironmentVariables` and `RuntimeEnvironmentSecrets` (if any) — these must be preserved in the update.

**Step 3 — Build the update JSON for backend service:**

Merge EXISTING variables with these NEW ones:

- `WAFR_DYNAMO_USERS_TABLE`: `"wafr-users"`
- `WAFR_DYNAMO_AUDIT_TABLE`: `"wafr-audit-log"`

RuntimeEnvironmentSecrets (new — from Secrets Manager):

- `WAFR_COGNITO_CLIENT_ID`: `"<secret-arn-for-wafr-cognito-client-id>"`

Use `aws apprunner update-service --cli-input-json file://backend-update.json --region us-east-1`. The JSON must include `ServiceArn` and the full `SourceConfiguration` block with merged env vars.

The `update-service` call triggers a redeployment. Verify both services return to RUNNING status. This may take 2-5 minutes.
