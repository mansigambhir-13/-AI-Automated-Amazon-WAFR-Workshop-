<task type="auto">
  <name>Task 1: Create wafr-sessions and wafr-review-sessions DynamoDB tables with GSIs and TTL</name>
  <files></files>
  <action>
Create the two session DynamoDB tables in us-east-1 using AWS CLI. Both use on-demand billing (PAY_PER_REQUEST).

**wafr-sessions table:**
- Partition key: `session_id` (S), Sort key: `created_at` (S)
- GSI: `user_id-created_at-index` — PK: `user_id` (S), SK: `created_at` (S), Projection: ALL
- Attribute definitions must include ALL three attributes: session_id, created_at, user_id (Pitfall 3 from research)
- After table is ACTIVE (`aws dynamodb wait table-exists`), enable TTL on attribute `expires_at`
- After table is ACTIVE, enable PITR via `update-continuous-backups`

**wafr-review-sessions table:**
- Partition key: `session_id` (S), Sort key: `item_id` (S)
- GSI: `status-created_at-index` — PK: `status` (S), SK: `created_at` (S), Projection: ALL
- Attribute definitions must include ALL four attributes: session_id, item_id, status, created_at
- After table is ACTIVE, enable TTL on attribute `expires_at`
- After table is ACTIVE, enable PITR

IMPORTANT: TTL and PITR require the table to be in ACTIVE status. Always use `aws dynamodb wait table-exists` before calling `update-time-to-live` or `update-continuous-backups` (Pitfall 2 from research).

Both tables get 365-day TTL via the `expires_at` attribute (per locked decision). The application code in Phase 2 will set the `expires_at` value at write time.
  </action>

- GSI: `email-index` — PK: `email` (S), no SK, Projection: ALL
- Attribute definitions: user_id, email
- After table is ACTIVE, enable PITR
- No TTL on this table (per locked decision)

**wafr-audit-log table:**
- Partition key: `user_id` (S), Sort key: `timestamp_session_id` (S) — NOTE: use underscore not hash; `#` is reserved in DynamoDB expression syntax (see Open Question 3 in research)
- GSI: `session_id-timestamp-index` — PK: `session_id` (S), SK: `timestamp` (S), Projection: ALL
- Attribute definitions must include ALL four attributes: user_id, timestamp_session_id, session_id, timestamp
- After table is ACTIVE, enable PITR
- No TTL on this table (per locked decision — audit trails kept indefinitely)

IMPORTANT: Use `aws dynamodb wait table-exists` before enabling PITR on each table.
