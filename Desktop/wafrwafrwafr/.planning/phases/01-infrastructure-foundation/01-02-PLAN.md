<task type="auto">
  <name>Task 1: Create Cognito User Pool with admin-only signup and password policy</name>
  <files></files>
  <action>
Create the Cognito User Pool in us-east-1 using AWS CLI.

**Pool configuration (all locked decisions — no alternatives):**
- Pool name: `wafr-user-pool`
- Admin-only signup: `--admin-create-user-config AllowAdminCreateUserOnly=true` (no self-service)
- Password policy: MinimumLength=12, RequireUppercase=true, RequireLowercase=true, RequireNumbers=true, RequireSymbols=true, TemporaryPasswordValidityDays=7
- MFA: Not required for v1 (per locked decision, deferred to AUTH-06)

**Command:**
```bash
aws cognito-idp create-user-pool \
  --pool-name wafr-user-pool \
  --admin-create-user-config AllowAdminCreateUserOnly=true \
  --policies 'PasswordPolicy={MinimumLength=12,RequireUppercase=true,RequireLowercase=true,RequireNumbers=true,RequireSymbols=true,TemporaryPasswordValidityDays=7}' \
  --region us-east-1
```

Capture the `UserPool.Id` from the response — it will be in format `us-east-1_XXXXXXXXX`. This ID is needed for subsequent commands and by Plan 01-03.
  </action>

<task type="auto">
  <name>Task 2: Create public App Client with SRP auth and user groups</name>
  <files></files>
  <action>
Using the User Pool ID from Task 1, create the App Client and both user groups.

**App Client configuration (locked decisions):**
- Client name: `wafr-app-client`
- No client secret: `--no-generate-secret` (public client, required for Amplify frontend)
- Access token validity: 1 hour (`--access-token-validity 1 --token-validity-units AccessToken=hours`)
- Auth flows: `ALLOW_USER_SRP_AUTH ALLOW_REFRESH_TOKEN_AUTH` — SRP is Amplify's default and most secure. Do NOT include `ALLOW_USER_PASSWORD_AUTH` (sends plaintext password) per research recommendation (Pitfall 5).

**Command:**
```bash
aws cognito-idp create-user-pool-client \
  --user-pool-id $POOL_ID \
  --client-name wafr-app-client \
  --no-generate-secret \
  --access-token-validity 1 \
  --token-validity-units AccessToken=hours \
  --explicit-auth-flows ALLOW_USER_SRP_AUTH ALLOW_REFRESH_TOKEN_AUTH \
  --region us-east-1
```

Capture `UserPoolClient.ClientId` from the response — needed by Plan 01-03 for Secrets Manager.

**User groups (locked decision):**
```bash
aws cognito-idp create-group \
  --user-pool-id $POOL_ID \
  --group-name WafrTeam \
  --description "Internal WAFR team members with full access" \
  --region us-east-1

aws cognito-idp create-group \
  --user-pool-id $POOL_ID \
  --group-name WafrClients \
  --description "External WAFR clients with limited access" \
  --region us-east-1
```

Record both the Pool ID and Client ID in the SUMMARY — Plan 01-03 depends on these values.
  </action>
  <verify>

  </verify>
