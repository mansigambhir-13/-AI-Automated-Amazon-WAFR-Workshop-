3. Create `wafr-agents/wafr/auth/jwt_middleware.py` implementing:
   - `HTTPBearer(auto_error=False)` — returns None instead of 403 when header absent (Pitfall 1 from research)
   - Lazy-initialized `PyJWKClient` singleton via `_get_jwks_client()` — avoids import-time KeyError if env vars not set (Pitfall 4 from research)
   - `verify_token(request: Request, credentials=Depends(_bearer)) -> dict` dependency:
     - If `AUTH_REQUIRED` env var is not `"true"` (case-insensitive), return synthetic claims `{"sub": "dev-user", "cognito:groups": ["WafrTeam"]}` — local dev bypass
     - If credentials is None, raise `HTTPException(status_code=401, detail="Missing or invalid authentication token")`
     - Decode token with `jwt.decode(token, signing_key.key, algorithms=["RS256"], issuer=COGNITO_ISSUER, options={"verify_exp": True, "verify_iss": True, "verify_signature": True, "require": ["exp", "iss", "sub", "token_use"]})`
     - Validate `token_use == "access"` and `client_id == APP_CLIENT_ID` (Cognito access tokens use `client_id` not `aud` — Pitfall 3)
     - On ANY exception, raise 401 with the generic message — never leak validation details
     - Store claims in `request.state.claims` so audit middleware (Plan 03) can read them
     - Return claims dict
   - `require_team_role(claims=Depends(verify_token)) -> dict` sub-dependency:
     - Check `cognito:groups` contains `"WafrTeam"`, else raise 403 "Requires WafrTeam role"
   - Environment variables used: `WAFR_COGNITO_USER_POOL_ID`, `WAFR_COGNITO_CLIENT_ID`, `AUTH_REQUIRED`
   - JWKS URL constructed as `https://cognito-idp.us-east-1.amazonaws.com/{pool_id}/.well-known/jwks.json`
   - Issuer constructed as `https://cognito-idp.us-east-1.amazonaws.com/{pool_id}`

2. Add `Depends(verify_token)` to ALL protected endpoints. Use `Depends(require_team_role)` for write/admin endpoints. Specifically:
   - **Public (NO auth dependency):** `GET /` (root), `GET /health`, any `/docs` auto-route
   - **Team-only (`Depends(require_team_role)`):** `POST /api/wafr/run`, `POST /api/wafr/process-file`, `DELETE /api/wafr/session/{id}`, `POST /api/wafr/session/{id}/cancel`, `POST /start` (async job start)
   - **Any authenticated user (`Depends(verify_token)`):** All other GET and POST endpoints (sessions, review, insights, questions, pillars, gaps, debug, reports, status, jobs)

   For each endpoint, add `claims: dict = Depends(verify_token)` or `claims: dict = Depends(require_team_role)` as a parameter. This makes claims available for audit logging (Plan 03). For endpoints that don't need claims in the handler body, use `dependencies=[Depends(verify_token)]` on the decorator instead.

   IMPORTANT: For endpoints that already have a `request: Request` parameter, add `claims` as an additional parameter. For endpoints without `request: Request`, add it now (needed for rate limiting in Plan 02 and for slowapi in Plan 02).
