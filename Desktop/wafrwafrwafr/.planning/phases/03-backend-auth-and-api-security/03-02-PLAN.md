Purpose: Prevents cross-origin abuse and protects the AI pipeline from excessive requests. CORS registered last in middleware stack ensures auth 401 responses carry CORS headers.
Output: `cors.py` and `rate_limit.py` modules, CORS middleware replaced, rate limit decorators on all endpoints.

2. Replace the existing CORS middleware block (currently `allow_origins=["*"]` with `allow_credentials=True` — this is a browser-rejected combination per Pitfall 6) with:
   ```python
   # Rate limiter middleware (registered before CORS — CORS must be outermost)
   app.state.limiter = limiter
   app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
   app.add_middleware(SlowAPIMiddleware)

   # CORS middleware — MUST be last add_middleware() call (executes first on requests)
   # This ensures auth 401 responses carry CORS headers (Pitfall 2 from research)
   app.add_middleware(
       CORSMiddleware,
       allow_origins=get_cors_origins(),
       allow_credentials=True,
       allow_methods=["GET", "POST", "DELETE", "OPTIONS"],
       allow_headers=["Authorization", "Content-Type", "Accept", "Cache-Control"],
       expose_headers=["X-Request-ID"],
       max_age=CORS_MAX_AGE,
   )
   ```
   CRITICAL ordering: SlowAPIMiddleware BEFORE CORSMiddleware in source code. CORSMiddleware is the last `add_middleware()` call, making it outermost (executes first on inbound requests). This ensures CORS headers are on ALL responses including 401 and 429.

3. Add `@limiter.limit()` decorators on endpoints per tier design. The decorator goes BELOW `@app.post/get/delete` (Pitfall: decorator must be inside the route decorator):
   - **Expensive tier (10/minute):**
     - `POST /api/wafr/run` — `@limiter.limit("10/minute")`
     - `POST /api/wafr/process-file` — `@limiter.limit("10/minute")`
     - `POST /start` (async job) — `@limiter.limit("10/minute")`
   - **Standard tier (60/minute):**
     - `POST /api/wafr/review/{session_id}/decision` — `@limiter.limit("60/minute")`
     - `POST /api/wafr/review/{session_id}/batch-approve` — `@limiter.limit("60/minute")`
     - `POST /api/wafr/review/{session_id}/finalize` — `@limiter.limit("60/minute")`
     - `POST /api/wafr/session/{session_id}/cancel` — `@limiter.limit("60/minute")`
     - `DELETE /api/wafr/session/{session_id}` — `@limiter.limit("60/minute")`
   - **Read tier (200/minute):** All GET endpoints — covered by `default_limits=["200/minute"]`, no decorator needed
   - **Excluded (no rate limit):** WebSocket `/ws/wafr/{session_id}` and SSE streaming responses — no decorator (WebSocket is not HTTP middleware; SSE is long-lived after the POST is already limited)

4. Ensure every rate-limited endpoint has `request: Request` as a parameter (Pitfall 5). Plan 03-01 should have added this, but verify and add if missing. slowapi reads the Request object to extract rate limit keys.
