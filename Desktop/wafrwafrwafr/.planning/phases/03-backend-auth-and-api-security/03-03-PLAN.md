<objective>
Implement audit trail logging that writes every authenticated API call to the wafr-audit-log DynamoDB table. A lightweight ASGI middleware captures metadata (user_id, path, method, IP, timestamp) for ALL requests, while per-endpoint BackgroundTasks add full request bodies for write operations.

Purpose: Provides a complete audit trail of who did what, when, and from where. Combined with the JWT auth from Plan 01, this enables security forensics and compliance.
Output: `audit.py` module with DynamoDB writer and ASGI middleware, wired into server.py with per-endpoint body logging on key write endpoints.
</objective>

  <files>

  <action>
1. Create `wafr-agents/wafr/auth/audit.py` with two components:

**Component A: `write_audit_entry()` function**
- Synchronous function (designed to run in BackgroundTasks thread pool or via asyncio.to_thread)
- Parameters: `user_id: str`, `session_id: Optional[str]`, `action_type: str`, `http_method: str`, `path: str`, `client_ip: str`, `request_body: Optional[dict] = None`, `status_code: Optional[int] = None`
- Constructs DynamoDB item:
  - PK `user_id`: from parameter
  - SK `timestamp_session_id`: `f"{iso_utc_timestamp}_{session_id or 'no-session'}"` (matches Phase 1 key schema: underscore separator, not hash)
  - Additional attributes: `session_id`, `timestamp`, `action_type`, `http_method`, `path`, `client_ip`, `request_body` (JSON-serialized string), `status_code`
- Uses lazy-initialized `boto3.resource("dynamodb", region_name="us-east-1")` singleton
- Table name from `WAFR_DYNAMO_AUDIT_TABLE` env var, defaults to `"wafr-audit-log"`
- Wraps `_audit_table.put_item(Item=item)` in try/except — on failure, logs warning via `logging.getLogger(__name__).warning()` but NEVER raises (fire-and-forget per locked decision)

**Component B: `AuditMiddleware` pure-ASGI class**
- `__init__(self, app)`: stores inner app
- `__call__(self, scope, receive, send)`:
  - If `scope["type"] != "http"`, pass through (WebSocket, lifespan)
  - Create `Request(scope, receive)` from starlette
  - Wrap `send` to capture response status code from `http.response.start` message
  - After response is sent, check `request.state.claims`:
    - If claims exist (authenticated request): schedule `write_audit_entry()` via `asyncio.get_event_loop().run_in_executor(None, write_audit_entry, ...)` with user_id from claims["sub"], path, method, client IP, and status code
    - If claims are None AND status code is 401: write a `"failed_auth"` entry with user_id="anonymous", capturing IP and path — per locked decision to log failed auth attempts
    - If claims are None AND status code is not 401 (public endpoints like /health): skip logging
  - Extract path from `scope["path"]`, method from `scope["method"]`, client IP from `scope["client"][0]` if available
  - The middleware does NOT capture request body (avoiding buffering large payloads). Request body logging happens per-endpoint via BackgroundTasks (Task 2).

2. Key design notes:
   - `asyncio.get_event_loop().run_in_executor(None, fn, ...)` runs sync boto3 in thread pool, non-blocking to event loop
   - This avoids `asyncio.create_task()` which would need an async wrapper around sync boto3
   - The middleware reads `request.state.claims` which was set by `verify_token` dependency (Plan 01) during request processing

  <name>Task 2: Wire audit middleware and per-endpoint body logging into server.py</name>
  <files>

  <action>
1. Add import at top of server.py:
   ```python
   from wafr.auth.audit import AuditMiddleware, write_audit_entry
   ```

2. Register AuditMiddleware in the middleware stack. It must be registered BEFORE SlowAPIMiddleware in source code (innermost in the stack). The final middleware registration order in server.py source should be:
   ```python
   # 1. Audit middleware (innermost — captures response status after all processing)
   app.add_middleware(AuditMiddleware)

   # 2. Rate limiter middleware
   app.state.limiter = limiter
   app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
   app.add_middleware(SlowAPIMiddleware)

   # 3. CORS middleware (outermost — MUST be last add_middleware call)
   app.add_middleware(CORSMiddleware, ...)
   ```
   Note: AuditMiddleware is a pure-ASGI class, not a Starlette BaseHTTPMiddleware, so it's compatible with `app.add_middleware()`.

3. Add per-endpoint `BackgroundTasks` audit writes with request body for key WRITE endpoints. On these endpoints, add `background_tasks: BackgroundTasks` parameter (if not already present) and add:
   ```python
   background_tasks.add_task(
       write_audit_entry,
       user_id=claims.get("sub", "unknown"),
       session_id=<session_id_from_route_or_body>,
       action_type="<action_name>",
       http_method="POST",
       path=request.url.path,
       client_ip=request.client.host if request.client else "unknown",
       request_body=<body>.model_dump(),
   )
   ```

   Endpoints that need per-endpoint body logging (the middleware handles metadata for ALL endpoints; these add request body):
   - `POST /api/wafr/run` — action_type="wafr_run", body=request body (RunWAFRRequest)
   - `POST /api/wafr/process-file` — action_type="process_file", body=request body
   - `POST /api/wafr/review/{session_id}/decision` — action_type="review_decision", body=request body
   - `POST /api/wafr/review/{session_id}/batch-approve` — action_type="batch_approve", body=request body
   - `POST /api/wafr/review/{session_id}/finalize` — action_type="finalize_review", body=request body
   - `DELETE /api/wafr/session/{session_id}` — action_type="delete_session", body=None (no body on DELETE)
   - `POST /start` — action_type="start_async_job", body=request body

   For these BackgroundTasks writes, the body logging duplicates what the middleware already captures (user_id, path, etc.) but adds the `request_body`. This is intentional — the middleware handles the base metadata for ALL 25+ endpoints, and per-endpoint tasks add the body for the important write operations. The audit table can hold both entries (different timestamps in SK make them unique rows).

   IMPORTANT: For the `POST /api/wafr/run` endpoint specifically, the `request_body` should EXCLUDE the `transcript` field if it exists (can be 500K chars). Use `body.model_dump(exclude={"transcript"})` and add a `transcript_length` field instead:
   ```python
   audit_body = body.model_dump(exclude={"transcript"})
   audit_body["transcript_length"] = len(body.transcript) if body.transcript else 0
   ```
   Same for `POST /start` and `POST /api/wafr/process-file` — exclude transcript/large fields, include length.

4. Update `wafr/auth/__init__.py` to export audit:
