**Part B: Wire REVIEW_STORAGE_TYPE in get_review_orchestrator()**

Find the `get_review_orchestrator()` function (or the singleton initialization pattern where `_review_orchestrator` is created). Currently it likely calls `create_review_storage("file")` or `create_review_storage("memory")`.

Update it to read the storage type from the environment:
```python

```

Pass `storage_type` to `create_review_storage(storage_type)`. This ensures:
- Local dev (no env var) uses file storage (safe default per research recommendation)
- App Runner (REVIEW_STORAGE_TYPE=dynamodb) uses DynamoDB

**Part C: Wire pipeline results storage**
