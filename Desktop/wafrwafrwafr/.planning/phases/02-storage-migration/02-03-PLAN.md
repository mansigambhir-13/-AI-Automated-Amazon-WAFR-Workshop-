<objective>
Build a standalone, idempotent migration script that reads existing file-based session data and pipeline results and writes them to DynamoDB via the DynamoDBReviewStorage class.

Purpose: Existing sessions stored as JSON files on disk will be lost on the next container restart. This script preserves them in DynamoDB before the storage switchover. Idempotency ensures it can be safely re-run during the migration window.
Output: `wafr-agents/scripts/migrate_sessions.py` — a CLI script with --dry-run, logging, and summary reporting.
</objective>

Create a new file `wafr-agents/scripts/migrate_sessions.py`. This is a standalone CLI script.

**Script structure:**

1. **Shebang and module docstring:**
   ```python
   #!/usr/bin/env python3
   """
   Migrate file-based review sessions and pipeline results to DynamoDB.

   Idempotent: safe to re-run — skips already-migrated sessions.
   Original files are never deleted or modified.

   Usage:
       python scripts/migrate_sessions.py                          # Full migration
       python scripts/migrate_sessions.py --dry-run                # Preview only
       python scripts/migrate_sessions.py --sessions-dir /path     # Custom path
       python scripts/migrate_sessions.py --log-file migration.log # Custom log
   """
   ```

2. **Imports:** `argparse`, `json`, `logging`, `sys`, `time` from stdlib. `from pathlib import Path`. Add `wafr-agents` to sys.path if needed so that `from wafr.storage.review_storage import create_review_storage` works.

3. **setup_logging(log_file)** function:
   - Configure root logger to write to both console (INFO level) and a log file (DEBUG level).
   - Log file path defaults to `migration_{timestamp}.log` in the current directory.
   - Return the logger.

4. **migrate_sessions(sessions_dir, storage, dry_run, logger)** function:
   - `sessions_dir`: Path to `review_sessions/sessions/`
   - Iterate over `sessions_dir.glob("*.json")`
   - For each file:
     - Extract `session_id` from filename (stem)
     - Check if session already exists in DynamoDB: `storage.load_session(session_id)`
     - If exists: log as "SKIP" and increment `skipped` counter
     - If not exists and not dry_run: read JSON, call `storage.save_session(data)`, increment `migrated` counter
     - If not exists and dry_run: log as "WOULD MIGRATE" and increment `migrated` counter (dry run count)
     - On exception: log ERROR with session_id and error message, increment `failed` counter, continue to next file
   - Return dict: `{"migrated": N, "skipped": N, "failed": N, "errors": [...]}`

5. **migrate_pipeline_results(pipeline_dir, storage, dry_run, logger)** function:
   - `pipeline_dir`: Path to `review_sessions/pipeline_results/`
   - Iterate over `pipeline_dir.glob("*.json")`
   - For each file:
     - Extract `session_id` from filename (stem)
     - Check if pipeline results already exist: `storage.load_pipeline_results(session_id)`
     - If exists: SKIP
     - If not exists and not dry_run: read JSON, call `storage.save_pipeline_results(session_id, data)`, increment counter
     - On exception: log ERROR, increment failed, continue
   - Return report dict same structure as above

6. **print_summary(session_report, pipeline_report, elapsed_seconds)** function:
   - Print a formatted summary table:
     ```
     ========================================
     MIGRATION SUMMARY
     ========================================
     Sessions:
       Migrated: X
       Skipped:  X (already in DynamoDB)
       Failed:   X

     Pipeline Results:
       Migrated: X
       Skipped:  X (already in DynamoDB)
       Failed:   X

     Total time: X.Xs
     ========================================
     ```
   - If any errors, print them after the summary.

7. **main()** function with argparse:
   - `--sessions-dir`: path to sessions directory (default: `review_sessions/sessions`)
   - `--pipeline-dir`: path to pipeline results directory (default: `review_sessions/pipeline_results`)
   - `--dry-run`: flag, preview without writing
   - `--log-file`: path to log file (default: auto-generated with timestamp)
   - Validate that both directories exist (exit with error if not)
   - Create DynamoDB storage via `create_review_storage("dynamodb")`
   - Call `migrate_sessions()` then `migrate_pipeline_results()`
   - Call `print_summary()`
   - Exit with code 1 if any failures, 0 otherwise

8. **if __name__ == "__main__": main()**

**Key implementation details:**
- Idempotency via check-then-write: call `load_session()` / `load_pipeline_results()` before writing. This is simpler than DynamoDB ConditionExpressions for multi-table writes.
- Never call `os.unlink()` or `Path.unlink()` on any file — original files are kept as backup per locked decision.
- The script must work when run from the `wafr-agents/` directory (relative paths for default directories).
- The `sys.path` manipulation (if needed) should prepend the parent of `wafr/` so that `from wafr.storage.review_storage import ...` works.
