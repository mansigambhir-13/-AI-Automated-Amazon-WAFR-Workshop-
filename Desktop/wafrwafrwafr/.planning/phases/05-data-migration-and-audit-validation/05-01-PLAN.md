**Create `wafr-agents/scripts/entrypoint.sh`:**

```bash
#!/bin/bash
set -e

echo "=== Starting uvicorn server ==="
cd /app
exec uvicorn wafr.ag_ui.server:app \
  --host 0.0.0.0 \
  --port 8000 \
  --timeout-keep-alive 300 \
  --workers 1 \
  --log-level info
```

Key design choices:
- `exec` replaces the shell process with uvicorn so signals (SIGTERM from App Runner) reach the server directly
- `cd /app` ensures correct WORKDIR even if entrypoint is invoked from elsewhere
- **No migration in entrypoint** — migration is run as a separate operator step via local `docker run` (Pattern B from research). This avoids embedding customer session data in the ECR image.

**Update `wafr-agents/Dockerfile`:**

1. Add `COPY scripts/ ./scripts/` AFTER the existing `COPY __init__.py .` line — this brings the entrypoint script into the image

2. Add `RUN chmod +x scripts/entrypoint.sh` after the COPY scripts line

3. **Do NOT add `COPY review_sessions/`** — customer session data must NOT be embedded in the Docker image pushed to ECR (security concern per checker). Migration runs locally with data mounted from the operator's machine.

4. Replace the existing multi-line CMD:
```dockerfile
CMD ["uvicorn", "wafr.ag_ui.server:app", \
     "--host", "0.0.0.0", \
     "--port", "8000", \
     "--timeout-keep-alive", "300", \
     "--workers", "1", \
     "--log-level", "info"]
```
With:
```dockerfile
CMD ["bash", "scripts/entrypoint.sh"]
```

The final Dockerfile should look like:
```dockerfile
FROM python:3.11-slim

ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1
ENV REVIEW_STORAGE_TYPE=dynamodb

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir --upgrade pip && \
    pip install --no-cache-dir -r requirements.txt && \
    pip install --no-cache-dir "uvicorn[standard]" "fastapi>=0.109.0"

COPY wafr/ ./wafr/
COPY knowledge_base/ ./knowledge_base/
COPY __init__.py .
COPY scripts/ ./scripts/
RUN chmod +x scripts/entrypoint.sh
RUN mkdir -p review_sessions/pipeline_results review_sessions/sessions /tmp/reports

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=5s --start-period=15s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1

CMD ["bash", "scripts/entrypoint.sh"]
```

Note: `COPY review_sessions/` is intentionally absent. Session data stays on the operator's machine and is migrated to DynamoDB via a local `docker run` command (see Task 2, Step 2). The `mkdir -p` line remains to create the empty directory structure the app may reference at runtime.

**Step 2 — Run migration locally via docker run (Pattern B):**

This runs the migration script from the operator's machine, mounting the local `wafr-agents/` directory which contains the `review_sessions/` data files. The data never enters the Docker image or ECR.

First, do a dry run to verify:
```bash
docker run --rm \
  -e AWS_ACCESS_KEY_ID="${AWS_ACCESS_KEY_ID}" \
  -e AWS_SECRET_ACCESS_KEY="${AWS_SECRET_ACCESS_KEY}" \
  -e AWS_SESSION_TOKEN="${AWS_SESSION_TOKEN}" \
  -e AWS_DEFAULT_REGION=us-east-1 \
  -e REVIEW_STORAGE_TYPE=dynamodb \
  -e WAFR_DYNAMO_SESSIONS_TABLE=wafr-sessions \
  -e WAFR_DYNAMO_REVIEW_SESSIONS_TABLE=wafr-review-sessions \
  -e WAFR_DYNAMO_USERS_TABLE=wafr-users \
  -e WAFR_DYNAMO_AUDIT_TABLE=wafr-audit-log \
  -v "$(pwd)/wafr-agents:/app" \
  -w /app \
  python:3.11-slim \
  bash -c "pip install boto3 && python scripts/migrate_sessions.py --dry-run"
```
